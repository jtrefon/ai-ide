Architecture

Two pipelines, one editor

Local and cloud are completely separate AI pipelines that share an editor. Each is optimized for what it does best - no compromises, no blended latency.

Local pipeline <100ms

Always on. Always private. No network required.

  • FIM inline completion - ghost text in <100ms
  • Inline AI popover - cursor-anchored Q&A
  • Semantic search - HNSW ANN, 10–50× faster
  • Direct local model chat (offline mode)
  • Model: Gemma 4 E4B IT 4-bit via MLX

Cloud pipeline Agentic

On demand. Full reasoning. Opt-in privacy.

  • Orchestration graph: Planner → Worker → QA → Final
  • Tool loop with stall detection + recovery
  • RAG context injection with evidence fusion ranking
  • 20+ tools - file I/O, terminal, search, project ops
  • Multi-provider: OpenRouter, DeepSeek, Alibaba, Kilo, local

For developers and teams

Local-by-default is a business advantage

Every AI IDE today sends your code to a cloud server before showing a suggestion. osx-ide reverses that assumption. The result is better for developers and the organizations they work for.

  • ✧ Completions in <100ms - no network jitter

Under the hood

Engineered for performance

MLX - Gemma 4 E4B

Apple MLX framework running a 4-bit quantized 4B model on the Neural Engine. 128K context window. <100ms end-to-end.

HNSW - FAISS

Hierarchical Navigable Small World index over FAISS vectors. 10–50× faster than brute-force with ~95–99% recall.

SQLite FTS5

Full-text search over indexed symbols, file content, and chunks. Combined with vector retrieval for grounded answers.

Tree-sitter - Neon

Tree-sitter AST parsing with the Neon runtime for fast, accurate syntax highlighting across 20+ languages.

EventBus

Typed, publisher-driven event system. Every pipeline, service, and UI component communicates through a shared bus - no tight coupling.

SwiftUI AppKit

Native NSTextView editor with SwiftUI overlays. No Chromium. No web view. Just native macOS, optimized for Apple Silicon.
