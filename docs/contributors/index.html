For contributors

osx-ide is MIT-licensed and built entirely in Swift. If you want to shape how native AI development works on macOS, this is the place.

Contribution guide Open issues GitHub

Codebase

~40,000 lines of Swift across osx-ide/, the test harness, and the UI layer. Build with Xcode 26+ and Apple Silicon. The project uses xcodebuild (not SwiftPM) for the main target, with SPM packages for SyntaxHighlighting (tree-sitter) and Terminal.

SwiftUI AppKit MLX FAISS SQLite Tree-sitter

Get started

./run.sh build
./run.sh test
./run.sh harness
xcodebuild test -scheme osx-ide

See AGENTS.md and ARCHITECTURE.md in the repo.

Where to start

Two-pipeline isolation

The core architecture: local (4B MLX, no orchestration) and cloud (orchestration graph, 20+ tools). They share zero AI execution code.

Services/LocalPipeline/ · Services/CloudPipeline/

Codebase index

SQLite FTS5 engine for symbols, chunks, files, and HNSW-backed embedding retrieval. All in Services/Index/.

Services/Index/DatabaseManager.swift

Inline completion engine

Local FIM completion with ghost-text rendering, caching, ranking, and sub-100ms latency. The tightest latency loop in the app.

Services/LocalPipeline/InlineCompletionEngine.swift

Agent tool system

20+ tools implementing the AITool protocol. The tool loop runs in ToolLoopHandler.swift with stall detection.

Services/Tools/ · Services/CloudPipeline/ToolLoopHandler.swift
