The problem

Current AI IDEs (Cursor, Copilot, Windsurf) are cloud-only by design. Every keystroke, every prompt, every completion requires a round-trip to a remote server. This means 500–2000ms latency for autocomplete, zero offline capability, no privacy guarantee for enterprise IP, and a $15–20/mo per-seat tax regardless of whether users actually need cloud models for every interaction.

Our moat

Dual-pipeline architecture

Local (<100ms, on-device LLM, offline) and cloud (orchestration graph, 20+ tools, RAG) pipelines share zero AI execution code. Each is independently optimized. This is not a feature switch - it's a deep architectural decision that competitors cannot retroactively match without a rewrite.

Native Apple Silicon

SwiftUI + AppKit, not Electron. MLX for on-device inference. Metal acceleration. Compass uses Apple Silicon as a first-class AI compute platform, not a thin client for someone else's server.

Business model aligned with architecture

The local pipeline is free and open source. The cloud pipeline is usage-based, not per-seat. Organizations pay only when they use cloud AI. This makes adoption frictionless and scales naturally with usage.

Privacy by design

Enterprise IP never leaves the endpoint without explicit consent. Compliance-ready from day one - not patched in later.
